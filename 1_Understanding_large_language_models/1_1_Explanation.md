# 📖 [심층 해설] 1장: 거대 언어 모델(LLMs)의 이해

우리가 배우게 될 책은 단순한 코딩 교본이 아닙니다. **"거대 언어 모델(Large Language Model, 이하 LLM)을 백지상태(From Scratch)에서부터 직접 만들어보는 것"**을 목표로 합니다. 우리가 흔히 사용하는 ChatGPT나 Claude와 같은 마법 같은 AI가 도대체 내부적으로 어떻게 동작하는지, 그 근원적인 수학과 구조를 파헤치는 여정의 첫걸음입니다.

## 1. 자연어 처리(NLP)의 진화와 LLM의 등장

책의 첫 문단은 LLM이 최근 몇 년간 개발되어 **자연어 처리(Natural Language Processing, NLP)** 분야에 새로운 시대를 열었다고 선언합니다. 그렇다면 '자연어 처리'란 무엇일까요?

### 💡 완전 기초: 자연어(Natural Language)란?
컴퓨터가 사용하는 언어(C, Python, Java 등)를 '인공어'라고 한다면, 우리 인간이 일상적으로 사용하는 한국어, 영어, 프랑스어 등을 **'자연어'**라고 부릅니다. 컴퓨터는 기본적으로 0과 1밖에 모르는 계산기입니다. 이런 계산기에게 "나 오늘 기분이 우울해, 위로가 되는 노래 좀 추천해 줄래?"라는 문장의 숨은 의도와 감정을 파악하게 만드는 모든 학문과 기술의 집합체를 **자연어 처리(NLP)**라고 합니다.

### 🚫 전통적인 방식의 한계 (LLM 이전 시대)
책에서는 LLM 등장 이전의 전통적인 방법(Traditional methods)들이 '스팸 메일 분류' 같은 작업에는 뛰어났다고 말합니다. 과거에는 어떻게 스팸을 걸러냈을까요? 
인간 전문가(개발자)가 직접 **'규칙(Rules)'**을 만들었습니다. 
- *규칙 1: 메일 제목에 "무료(free)", "당첨(win)"이라는 단어가 있으면 스팸이다.*
- *규칙 2: 느낌표(!!!)가 3개 이상 연속으로 쓰이면 스팸이다.*

이런 방식을 **규칙 기반 시스템(Rule-based system)** 또는 초기 머신러닝의 **수동 피처 추출(Manual Feature Extraction)** 방식이라고 합니다. 이는 "이것은 A인가, B인가?"를 분류(Categorization)하는 데는 꽤 유용했습니다. 

**하지만 치명적인 단점이 있었습니다.**
과거의 모델에게 "키워드 3개(회의, 금요일, 취소)를 줄 테니 정중한 이메일을 한 통 작성해 줘."라고 명령하면 완전히 고장 나버렸습니다. 왜냐하면 인간의 언어는 단순한 단어의 조합이 아니라, **'문맥(Context)', '뉘앙스(Nuance)', '어조(Tone)'**가 복잡하게 얽혀있기 때문입니다. 수만 가지의 문법과 예외 상황을 인간이 일일이 규칙으로 코딩하는 것은 불가능에 가깝습니다. 즉, 복잡한 이해와 '새로운 텍스트의 생성(Generation)' 작업에서 전통적 모델은 참담히 실패했습니다.

## 2. LLM은 과연 인간의 언어를 '이해(Understand)'하는가?

책에서 아주 중요하게 짚고 넘어가는 철학적이고 기술적인 포인트가 있습니다.
> *"언어 모델이 언어를 '이해'한다고 말할 때, 이는 모델이 사람처럼 의식이나 진짜 깨달음을 가지고 있다는 뜻이 아닙니다. 단지 상황에 맞게 텍스트를 처리하고 생성하는 능력이 일관성 있고 문맥상 적절해 '보인다(appear)'는 뜻입니다."*

### 💡 완전 기초: 확률적 앵무새(Stochastic Parrot)와 다음 단어 예측
우리가 챗봇에게 질문을 던졌을 때, 챗봇이 완벽한 논리로 대답하는 것을 보면 마치 안에 진짜 사람이 들어있는 것 같습니다. 하지만 내부적으로 컴퓨터는 의미를 깨닫는 것이 아닙니다. 오직 **수학적인 통계와 확률**을 계산할 뿐입니다.

예를 들어, **"나는 아침에 일어나서 밥을 ____"** 이라는 문장이 주어졌을 때, 
- '먹었다'가 올 확률: 85%
- '버렸다'가 올 확률: 10%
- '날았다'가 올 확률: 0.01%

LLM은 자신이 학습한 수조 개의 인터넷 글들을 바탕으로, 주어진 문맥 뒤에 **어떤 단어가 오는 것이 수학적으로 가장 자연스러운지(확률이 높은지)를 끊임없이 예측**하여 이어 붙이는 기계입니다. 이렇게 단어를 하나씩 확률적으로 이어 붙이다 보니, 최종적으로 완성된 문장이 마치 인간이 깊이 생각하고 쓴 글처럼 '보이는' 것입니다. 

따라서 LLM의 '이해력'은 자아성찰의 결과가 아니라, **압도적인 데이터가 만들어낸 정교한 통계적 패턴 인식의 결과**입니다.

## 3. 세상을 바꾼 두 가지 열쇠: 트랜스포머(Transformer)와 방대한 데이터

LLM이 기존의 NLP 모델들을 압도하게 된 성공 요인으로 책은 두 가지를 꼽습니다.

1. **트랜스포머(Transformer) 아키텍처**
   - 2017년 구글(Google)의 논문 "Attention Is All You Need"에서 처음 발표된 혁신적인 신경망 구조입니다.
   - 기존 모델들은 책을 읽을 때 단어를 왼쪽부터 오른쪽으로 하나씩 순서대로 읽고 까먹는 구조였습니다(RNN, LSTM). 문장이 길어지면 앞부분을 잊어버리는 치명적 문제가 있었죠.
   - **트랜스포머**는 다릅니다. 문장 내의 모든 단어를 한꺼번에 펼쳐놓고, **어떤 단어가 어떤 단어와 깊은 연관(Attention)이 있는지를 집중적으로 파악**합니다. 
   - 예를 들어, *"그 소녀는 강아지를 쫓아갔다. 왜냐하면 **그것**이 뼈다귀를 물고 도망갔기 때문이다."* 라는 문장에서, 트랜스포머는 수학적인 계산을 통해 **'그것'**이 '소녀'가 아니라 **'강아지'**라는 것을 문맥적으로 정확히 짚어냅니다. (이것을 Self-Attention 메커니즘이라고 하며, 책의 3장에서 직접 코딩하게 됩니다.)

2. **방대한 훈련 데이터(Vast amounts of data)**
   - 트랜스포머라는 훌륭한 두뇌 구조가 생겼으니, 이제 지식을 채워 넣을 차례입니다. LLM은 인터넷에 존재하는 위키백과, 뉴스 기사, 책, 블로그 포스트, 코드 등 인류가 남긴 텍스트 데이터의 거의 전부를 읽어 들입니다. 데이터가 많아질수록 모델은 인간 언어의 미묘한 차이, 맥락, 패턴을 스스로(수동 규칙 없이) 파악하게 됩니다.

# 📖 [심층 해설] 1.1 What is an LLM? (LLM이란 무엇인가?)

이제 본격적으로 "거대 언어 모델"이라는 이름 자체를 해부해 보겠습니다.

### "Large (거대한)"의 두 가지 의미
1. **데이터 셋의 크기 (Immense dataset)**: 앞서 말했듯 인터넷에 공개된 텍스트 전체에 맞먹는 상상 초월의 데이터를 학습합니다.
2. **파라미터(Parameters)의 개수**: 모델 크기가 거대하다는 뜻입니다. 현대의 LLM은 수백억 개에서 수조 개의 파라미터를 가집니다.

### 💡 완전 기초: 파라미터(Parameter)란 무엇인가요?
우리 인간의 뇌는 수백억 개의 '뉴런(신경세포)'이 서로 연결되어 있고, 그 연결 부위를 '시냅스'라고 부릅니다. 우리가 무언가를 학습하면 이 시냅스의 연결 강도가 굵어지거나 얇아지면서 기억이 저장됩니다.

인공지능도 똑같습니다. 인공 신경망 안에는 무수히 많은 인공 뉴런들이 거미줄처럼 연결되어 있습니다. 이 **연결의 강도를 숫자로 표현한 것**이 바로 **'파라미터(가중치, Weights)'**입니다.
- *"사과"*라는 단어와 *"맛있다"*라는 단어 사이의 신경망 연결선 가중치: `9.8`
- *"사과"*라는 단어와 *"노트북"*이라는 단어 사이의 신경망 연결선 가중치: `0.1`

LLM은 수천억 개의 파라미터 값을 가지고 있습니다. 모델을 '훈련(Training)'시킨다는 것은, 수많은 책을 읽히면서 **"다음 단어를 완벽하게 예측할 수 있도록 저 수천억 개의 파라미터 숫자들을 이리저리 조율하고 최적화(Optimize)하는 과정"**을 뜻합니다. 

이 **"다음 단어 예측(Next-word prediction)"**이라는 개념이 바로 LLM의 알파이자 오메가입니다. 
단순히 다음 단어 하나 맞추는 게임을 수조 번 반복시켰을 뿐인데, 모델은 단어의 순서, 문법, 문맥, 심지어는 상식과 논리, 코딩하는 법까지 깨닫게 됩니다. 연구자들조차 이렇게 단순한 학습 방법이 이토록 경이로운 지능을 만들어낼 줄은 처음엔 상상하지 못했다고 책은 묘사합니다.

**1. 전통적인 머신러닝(Machine Learning)의 방식**
전통적인 ML 모델에 이메일 데이터를 넣기 전에, **반드시 인간 전문가의 개입이 필요**했습니다.
- 인간: "기계야, 메일을 통째로 주면 네가 이해를 못 할 테니, 내가 중요한 **'특징(Feature)'**을 뽑아줄게."
- 인간이 추출한 특징: 
  1) 대문자가 몇 번 쓰였나? 
  2) '무료'라는 단어가 있는가? 
  3) URL 링크가 포함되어 있는가?
- 기계: 인간이 요약해 준 이 숫자 데이터(특징)들만 보고, "특징 1, 2, 3이 높으면 스팸이네!"라고 학습합니다.
- 이를 **수동 피처 추출(Manual feature extraction)**이라고 합니다. 인간의 직관에 크게 의존합니다.

**2. 딥러닝(Deep Learning)의 방식**
- 인간: "기계야, 복잡하게 특징 같은 거 내가 안 뽑아줄게. 그냥 이메일 원문 텍스트 통째로 100만 개 던져줄 테니까 네가 알아서 스팸인지 아닌지 구별해 봐."
- 기계(딥러닝): 인공신경망의 여러 층(Layers)을 거치면서, 얕은 층에서는 단어 하나하나를 인식하고, 깊은 층으로 갈수록 단어들의 조합, 문장 전체의 뉘앙스를 **스스로(자동으로) 파악**합니다.
- 즉, 인간이 어떤 특징이 중요한지 알려줄 필요 없이 **네트워크가 데이터 속에서 스스로 복잡한 추상화(Abstractions)와 패턴을 찾아냅니다.**

**3. LLM과 GenAI의 관계**
LLM은 딥러닝의 하위 분야입니다. 텍스트를 처리하는 데 특화되어 있죠. 
그리고 **생성형 AI(Generative AI, GenAI)**는 데이터를 바탕으로 무언가 '새로운 콘텐츠'를 만들어내는 AI를 뜻합니다. 그림을 그리는 AI(DALL-E), 음악을 만드는 AI도 모두 GenAI입니다.
우리가 아는 ChatGPT는 언어(텍스트)를 생성해 내기 때문에 **LLM이자 동시에 GenAI인 교집합**에 속하게 됩니다.

# 🚀 [2027년 이후 최신 실무 트렌드] 현업에서는 어떻게 사용될까?

책의 기초 개념을 2027년 이후의 최신 실무 현장에 적용하면 어떻게 진화하는지 설명해 드리겠습니다. 단순한 텍스트 분류를 넘어서 현재 업계는 완전히 새로운 차원으로 LLM을 활용하고 있습니다.

### 실전 예시 1: RAG (검색 증강 생성, Retrieval-Augmented Generation) 기반의 사내 전문가 시스템
* **개념**: 책에서 말하는 LLM은 학습된 시점까지의 데이터만 압니다. 실무에서는 회사 내부의 최신 기밀문서나 가이드라인을 LLM에게 가르쳐야 합니다. 하지만 매번 수천억 개의 파라미터를 다시 학습(Fine-tuning)시키는 것은 천문학적인 비용이 듭니다.
* **실무 활용**: 2027년 현업에서는 **RAG(검색 증강 생성)** 기술이 표준입니다.
  1. 사용자가 질문합니다. ("우리 회사 2027년 3분기 환불 규정이 뭐야?")
  2. 시스템이 회사 사내 데이터베이스를 '검색(Retrieval)'하여 관련 문서를 텍스트로 찾아옵니다.
  3. 이 문서를 LLM의 프롬프트(명령창)에 몰래 밀어 넣습니다.
  4. LLM은 찾아온 문서를 '읽고(이해하고)' 그 기반 위에서 텍스트를 '생성(Generation)'하여 대답합니다.
  이로 인해 할루시네이션(환각 현상: 거짓말을 지어내는 것)이 극단적으로 줄어들고, 수동 피처 추출 없이도 완벽한 비즈니스 로직을 처리합니다.

### 실전 예시 2: Agentic Workflow (자율 에이전트 워크플로우)
* **개념**: 과거에는 LLM이 그저 묻는 말에 답만 하는 '수동적인 챗봇'이었습니다.
* **실무 활용**: 현재는 LLM이 스스로 생각하고 도구를 사용하는 **'에이전트(Agent)'**로 발전했습니다.
  예를 들어, 스팸 필터를 넘어선 **사내 보안 시스템**을 생각해 봅시다.
  과거 머신러닝 스팸 필터는 단순히 메일을 차단할 뿐이었습니다. 하지만 현재의 LLM 에이전트는 다음과 같이 행동합니다.
  - LLM이 의심스러운 이메일을 감지합니다. (딥러닝 패턴 인식)
  - LLM이 스스로 사내 서버의 터미널(Terminal) 도구를 호출하여 발송자의 IP를 역추적(run_command)합니다.
  - LLM이 피싱 사이트 URL을 분석하고(search_web 도구 사용), 코드를 해석하여 악성 스크립트인지 확인합니다.
  - 최종 판단 후, 보안 팀의 슬랙(Slack)으로 요약된 보고서를 생성(GenAI)하여 발송합니다.
  단순한 '다음 단어 예측' 모델에 **'도구 사용 권한'과 '계획을 세우는 능력(Planning)'**을 부여하여 인간 직원을 대체하는 실무 자동화 시스템을 구축하는 것이 최신 트렌드입니다.

### 실전 예시 3: On-Device AI와 경량화 (SLM)
* **개념**: "Large(거대한)" 모델은 구동하는 데 비싼 GPU 서버가 필요합니다.
* **실무 활용**: 스마트폰이나 IoT 기기 내부에서 인터넷 연결 없이 작동하는 **SLM (Small Language Model)**이 대세가 되었습니다. 파라미터가 수천억 개가 아니라 수십억 개 수준으로 작지만, 특정 목적(예: 폰 내부의 개인 비서, 자동 번역기)에 특화되도록 매우 정교하게 훈련시킵니다. 앞으로 우리가 책에서 직접 코딩해 볼 모델도 바로 이 맥락처럼 개인이 충분히 다룰 수 있는 크기의 커스텀 언어 모델을 만드는 과정이 될 것입니다.

전통적인 머신러닝(Traditional Machine Learning)과 딥러닝(Deep Learning)이 스팸 필터를 만들 때 데이터를 처리하는 가장 결정적인 차이점은 무엇인가요?

LLM이 인간의 언어를 '이해'하고 논리적인 글을 써 내려가는 것처럼 보일 때, 그 내부에서 실제로 벌어지고 있는 단 하나의 단순한 핵심 메커니즘은 무엇인가요?

### ❓ 질문 3
ChatGPT는 인공지능 분야의 벤다이어그램(Figure 1.1)에서 어떤 개념들에 속하며, 왜 '생성형 AI(Generative AI)'라고도 불리나요?

---

### 💡 모범 답안 (정확한 이해를 위한 체크)

**답안 1**
가장 큰 차이는 **수동 피처 추출(Manual Feature Extraction)의 유무**입니다. 전통적인 머신러닝은 인간 전문가가 데이터(이메일)에서 중요한 특징(예: '당첨' 단어 빈도, 대문자 개수 등)을 미리 뽑아내어 기계에게 줘야만 학습할 수 있었습니다. 반면, 딥러닝은 여러 층의 신경망을 통해 원시 데이터(이메일 텍스트 원문)로부터 **스스로 중요한 특징과 패턴을 찾아내어(추상화하여)** 학습합니다.

**답안 2**
그 핵심 메커니즘은 바로 **'다음 단어 예측(Next-word prediction)'**입니다. LLM은 자아나 의식을 가진 것이 아니라, 방대한 데이터 학습을 통해 수천억 개의 파라터를 최적화하여, 주어진 문맥 바로 다음에 등장할 가장 확률이 높은(통계적으로 자연스러운) 단어를 끊임없이 이어 붙이는 수학적 계산을 수행할 뿐입니다.

**답안 3**
ChatGPT는 인공지능(AI)의 하위 개념인 머신러닝(ML), 그 안의 딥러닝(DL), 그리고 텍스트에 특화된 거대 언어 모델(LLM)에 모두 속합니다. 또한, 기존에 존재하는 정답을 골라내는 분류(Categorization) 작업이 아니라, 사용자 요청에 맞춰 세상에 없던 **새로운 텍스트 문장을 '창작(Generate)'해내기 때문**에 생성형 AI(Generative AI)의 교집합에 속한다고 부릅니다.

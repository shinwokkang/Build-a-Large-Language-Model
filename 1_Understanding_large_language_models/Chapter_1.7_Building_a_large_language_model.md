# 📖 [심층 해설] 1.7: 거대 언어 모델 구축하기 (Building a large language model) & 1장 총정리

1장의 마지막 섹션에 도달했습니다! 지금까지 우리는 LLM이 무엇인지, 왜 놀라운지, 그리고 어떤 원리와 뼈대(트랜스포머, GPT)로 움직이는지 이론적으로 튼튼하게 무장했습니다. 이제 이론을 넘어, **앞으로 이 책을 통해 우리가 직접 코드로 구현하게 될 "LLM 구축의 3단계 여정"**을 전체적인 조감도로 살펴보겠습니다.

첨부해주신 Figure 1.9는 우리가 파이토치(PyTorch) 코드를 한 줄 한 줄 짜며 완성해 나갈 전체 로드맵입니다. 마치 빈 땅에 설계도를 그리고, 건물을 올리고, 인테리어를 하는 과정과 같습니다.

### 🏗️ STAGE 1: 뼈대 만들기 (Building an LLM)
가장 먼저 아무것도 없는 백지상태에서 AI의 뇌(뇌세포와 신경망)를 물리적으로 조립하는 단계입니다.
1. **데이터 준비 및 샘플링 (Data preparation & sampling)**: 
   - 앞서 배운 방대한 텍스트 데이터를 컴퓨터가 소화할 수 있는 '토큰(Token)' 단위로 잘게 쪼개는 작업(Tokenization)을 코딩합니다.
2. **어텐션 메커니즘 (Attention mechanism)**: 
   - 트랜스포머의 심장부입니다. 단어와 단어 사이의 관계와 문맥을 파악하는 수학 공식(행렬 곱셈)을 코딩합니다. (가장 수학적으로 아름답고 복잡한 3장의 내용입니다.)
3. **LLM 아키텍처 (LLM architecture)**: 
   - 어텐션 부품들을 이리저리 조립하여, GPT와 똑같이 작동하는 '디코더 전용 신경망 뼈대'를 최종 완성합니다. 이 시점의 모델은 뇌 구조는 완벽하지만, 아직 아무런 글도 읽지 않은 신생아 상태입니다.

### 🧠 STAGE 2: 뇌 발달시키기 (Foundation model)
신생아에게 세상의 온갖 지식을 주입하여 똑똑한 어른(기반 모델)으로 키우는 단계입니다.
1. **사전 학습 (Pretraining)**: 
   - 완성된 뼈대에 앞서 준비한 텍스트를 부어 넣고 "다음 단어 예측해 봐!"라는 훈련 루프(Training loop)를 코딩하여 수없이 반복시킵니다.
2. **모델 평가 (Model evaluation)**: 
   - 이 모델이 헛소리를 하는지, 아니면 진짜 말이 되는 문장을 쓰는지 수학적으로 채점하는 코드를 만듭니다.
3. **[2027년 실무 핵심] 사전 학습된 가중치 불러오기 (Load pretrained weights)**: 
   - 우리가 직접 GPT-3 급의 똑똑한 모델을 처음부터 훈련(Pretraining)시키려면 수십억 원의 전기세와 슈퍼컴퓨터가 필요합니다. 따라서 책에서는 교육 목적으로 아주 작은 데이터로 훈련 원리만 배운 뒤, 실무에서 하듯 **오픈소스로 공개된 똑똑한 모델의 뇌 구조(Weights)를 무료로 다운받아 우리의 뼈대에 이식하는 방법**을 배웁니다.

### 🎯 STAGE 3: 내 맘대로 커스텀하기 (Fine-tuning)
똑똑해진 기반 모델을 '나만의 목적'에 맞게 개조(인테리어)하는 단계입니다.
1. **분류기 (Classifier)**: 
   - 긍정/부정, 스팸/정상 등을 100% 완벽하게 구별해 내는 모델로 미세 조정합니다. (예: 사내 보안용 스팸 필터 제작)
2. **개인 비서 (Personal assistant)**: 
   - 정답(Label)이 달린 '질문-답변(Instruction)' 세트를 훈련시켜, 묻는 말에 친절하게 대답하는 나만의 커스텀 ChatGPT를 만들어 냅니다.

## 2. 🚀 [2027년 최신 실무 관점] 실무자들은 이 3단계를 어떻게 쓰는가?

이 책의 로드맵은 완벽하지만, **2027년의 실무 현장**에서는 이 3단계 중 어디에 가장 많은 힘을 쏟을까요? 

정답은 **오직 STAGE 3 (Fine-tuning)**입니다.
* STAGE 1과 2는 메타(Meta), 구글(Google), 미스트랄(Mistral) 같은 자본력이 빵빵한 글로벌 빅테크 기업들의 전쟁터입니다. 그들이 천문학적인 돈을 들여 STAGE 2까지 완성된 모델(Llama, Gemma 등)을 무료로 공개합니다.
* **우리 같은 개인 개발자나 기업의 AI 엔지니어들은 빅테크가 던져준 STAGE 2의 모델을 주워와서, STAGE 3 (우리 회사만의 데이터를 입히는 미세 조정 작업)에 모든 역량을 집중합니다.**
* 그럼에도 불구하고 우리가 이 책을 통해 STAGE 1과 2를 밑바닥부터 짜보는 이유는, **엔진의 구조를 알아야 자동차를 내 마음대로 완벽하게 튜닝(STAGE 3)할 수 있기 때문**입니다. 내부 구조를 모른 채 프롬프트만 만지작거리는 겉핥기식 개발자와, 어텐션 수식을 이해하고 모델의 뇌수를 직접 만질 줄 아는 엔지니어의 몸값과 실력 차이는 여기서 발생합니다.

## 3. 🎯 1장 총정리 (Summary)

지금까지 배운 1장의 거대한 개념들을 실무자의 언어로 완벽히 요약해 보겠습니다.

1. **패러다임의 전환**: 언어 처리는 인간이 규칙을 짜던 시대(Rule-based)에서, 기계가 문맥을 씹어 먹는 딥러닝(Deep Learning)의 시대로 진화했습니다.
2. **트랜스포머의 승리**: 모든 것의 중심에는 문장 전체의 상관관계를 파악하는 **어텐션(Attention) 메커니즘**이 있습니다.
3. **GPT의 선택**: 굳이 복잡하게 인코더+디코더를 다 쓰지 않고, 오로지 글을 지어내는 **'디코더(Decoder)' 하나만 뚝 떼어내어 극대화**한 것이 바로 GPT(그리고 현대의 거의 모든 LLM)입니다.
4. **학습의 2단계**: 
   - 1단계(사전 학습): 엄청난 양의 라벨 없는 인터넷 글로 "다음 단어 맞추기"를 시킵니다. (천문학적 비용 발생, 기반 모델 완성)
   - 2단계(미세 조정): 소량의 라벨 달린 데이터로 "질문에 대답하기"를 가르칩니다. (비용 저렴, 챗봇 완성)
5. **창발적 지능 (Emergence)**: 1단계에서 그저 다음 단어만 맞추라고 훈련시켰을 뿐인데, 데이터가 워낙 거대하다 보니 모델 스스로 번역, 요약, 분류, 코딩하는 법을 스스로 깨우쳐 버렸습니다. 이것이 LLM의 가장 무서운 점입니다.

# 📝 1장 최종 이해도 점검 (Verification Q&A)

1장을 훌륭하게 완주하셨습니다! 다음 세 가지 질문을 통해 1장 전체의 흐름을 마지막으로 점검해 보시기 바랍니다.

### ❓ 질문 1
현대의 LLM 엔지니어들은 회사에서 AI 비서를 만들 때, STAGE 1(아키텍처 구축)과 STAGE 2(사전 학습)를 처음부터 직접 수행하지 않는 경우가 대부분입니다. 그 이유가 무엇이며, 대신 어떤 방식을 취하는지 '가중치(Weights)'라는 단어를 포함하여 설명해 보세요.

### ❓ 질문 2
트랜스포머 아키텍처의 핵심 부품인 **'어텐션 메커니즘(Attention Mechanism)'**이 기존의 방식(예: 순서대로 단어를 읽는 방식)에 비해 압도적으로 문맥을 잘 파악하는 이유를 직관적으로 설명해 보세요.

### ❓ 질문 3
우리가 흔히 쓰는 ChatGPT는 1단계(사전 학습) 모델인가요, 아니면 2단계(미세 조정) 모델인가요? 만약 1단계 모델에게 말을 걸었을 때와 2단계 모델에게 말을 걸었을 때, 답변의 뉘앙스가 어떻게 다를지 유추해 보세요.

### 💡 모범 답안 (정확한 이해를 위한 체크)

**답안 1**
STAGE 2(사전 학습)를 수행하기 위해서는 수십억 원의 클라우드 컴퓨팅 비용과 수백억 장의 데이터가 필요하기 때문입니다. 따라서 실무에서는 빅테크 기업들이 이미 학습을 끝내고 오픈소스로 공개해 둔 모델의 뇌 구조인 **'사전 학습된 가중치(Pretrained Weights)'**를 무료로 다운로드하여 뼈대에 얹은 후, 우리 회사의 데이터만 살짝 얹어서 훈련시키는 STAGE 3(미세 조정)만을 수행하여 효율적으로 AI를 구축합니다.

**답안 2**
기존 모델들은 문장을 읽을 때 앞단어부터 순서대로 읽고 기억력이 점차 흐려졌습니다. 하지만 '어텐션 메커니즘'은 문장 내의 모든 단어를 한꺼번에 펼쳐놓고, **특정 단어가 문장 내의 다른 어떤 단어들과 가장 깊은 연관(집중, Attention)이 있는지 수학적으로 동시에 계산**합니다. 덕분에 멀리 떨어져 있는 단어들 간의 관계도 놓치지 않고 얽힌 문맥을 완벽하게 캐치할 수 있습니다.

**답안 3**
ChatGPT는 2단계인 **미세 조정(Fine-tuning, 특히 Instruction Fine-tuning)**까지 완료된 모델입니다. 만약 1단계 사전 학습만 끝난 모델(기반 모델)에게 "사과의 효능이 뭐야?"라고 묻는다면, 대답하려는 의도 없이 그저 텍스트를 이어가기 위해 "...라고 많은 사람들이 검색엔진에 묻습니다." 식으로 평범한 기사 문맥을 이어갈 것입니다. 반면 2단계 미세 조정까지 끝난 ChatGPT에게 물으면 "사과의 효능은 다음과 같습니다. 1..." 이라며 묻는 말에 '대답'하는 비서로서의 정해진 패턴을 따르게 됩니다.

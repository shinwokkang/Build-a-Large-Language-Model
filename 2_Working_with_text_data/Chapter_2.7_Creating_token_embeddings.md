# 📖 [심층 해설] 2.7: 토큰 임베딩 만들기 (Creating token embeddings) 완벽 정복

우리는 앞선 과정에서 텍스트를 BPE 토크나이저로 잘라 정수 번호(Token ID)로 바꾸고, 이를 슬라이딩 윈도우 방식으로 잘라 파이토치 텐서(Tensor)에 담는 것까지 성공했습니다. 

이제 데이터 전처리의 **최종 관문**입니다. 파이토치 텐서 안에 들어 있는 `[2, 3, 5, 1]` 같은 뻣뻣한 정수들을 신경망이 소화할 수 있는 유연한 실수 형태의 **다차원 벡터(Embedding Vector)**로 변환하는 과정을 코드로 완벽하게 파헤쳐 보겠습니다.

---

## 1. 🧠 왜 '임베딩(Embedding)'이 필수적인가? (Figure 2.15 해설)

<img width="685" height="510" alt="Figure 2 15" src="https://github.com/user-attachments/assets/7c244fa0-c44f-45c2-b43b-8b25872ff66c" />


그림 2.15를 보면 텍스트가 토큰 ID를 거쳐 최종적으로 네모난 칸 여러 개(벡터)로 변환된 뒤에야 비로소 GPT 모델 안으로 들어갑니다. 그렇다면 굳이 왜 번거롭게 정수를 실수 벡터로 또 바꿔야 할까요?

1. **정수(Discrete)의 한계**: 
   - 사과가 1번, 바나나가 2번, 강아지가 10,000번이라고 가정해 봅시다. 정수 자체에는 아무런 의미적 관계가 없습니다. "바나나가 사과보다 2배 더 가치가 있다"거나 "강아지는 사과보다 10,000배 더 크다"는 뜻이 전혀 아닙니다. 
2. **미분 불가능성**: 
   - 딥러닝 모델(신경망)은 역전파(Backpropagation)라는 수학적 미분 알고리즘을 통해 에러를 수정하며 학습합니다. 정수는 1 다음이 바로 2로 건너뛰어 버려서 그사이에 소수점 미분이 불가능합니다.
3. **임베딩(Continuous Vector)의 마법**: 
   - 정수를 소수점이 있는 다차원 실수 벡터(예: `[0.34, -1.2, 0.8]`)로 바꾸면, 단어들을 공간상에 둥둥 떠 있는 '좌표'로 만들 수 있습니다. 
   - 학습이 진행될수록 비슷한 의미의 단어들(사과, 바나나)은 우주 공간에서 서로 가까운 좌표로 모이게 되고, 이 거리를 미분하여 미세하게 조정할 수 있게 됩니다.

---

## 2. 💻 코드로 보는 임베딩 계층(Layer)의 탄생

이 복잡한 수학적 공간을 파이토치(PyTorch)에서는 `torch.nn.Embedding` 이라는 함수 단 하나로 만들어냅니다.

### 📝 코드 1: 임베딩 뼈대 세우기
```python
import torch

# 1. 상황 가정: 입력으로 들어온 토큰 ID들
input_ids = torch.tensor([2, 3, 5, 1])

# 2. 하이퍼파라미터 설정
vocab_size = 6    # 우리가 가진 단어 사전의 총 개수가 딱 6개뿐이라고 가정
output_dim = 3    # 단어 하나를 몇 차원(몇 개의 숫자)으로 표현할 것인가? (여기선 3차원)

# 3. 재현성을 위한 시드 고정 (언제 실행해도 똑같은 랜덤 값이 나오게 함)
torch.manual_seed(123)

# 4. 임베딩 계층 생성
embedding_layer = torch.nn.Embedding(vocab_size, output_dim)
print(embedding_layer.weight)
```
* **💡 실전 비교**: 예제에서는 이해를 돕기 위해 차원(output_dim)을 3으로 잡았지만, 실제 GPT-3 모델은 단어 하나를 무려 **12,288차원**의 거대한 벡터로 표현합니다. (단어 하나의 뉘앙스를 1만 2천 개의 숫자로 쪼개서 담는다는 뜻입니다.)

### 📝 출력 결과 분석 (가중치 행렬)
`print(embedding_layer.weight)`의 결과를 보면 엄청난 사실을 알 수 있습니다.
```text
Parameter containing:
tensor([[ 0.3374, -0.1778, -0.1690],  <-- 0번 단어의 벡터
        [ 0.9178,  1.5810,  1.3010],  <-- 1번 단어의 벡터
        [ 1.2753, -0.2010, -0.1606],  <-- 2번 단어의 벡터
        [-0.4015,  0.9666, -1.1481],  <-- 3번 단어의 벡터
        [-1.1589,  0.3255, -0.6315],  <-- 4번 단어의 벡터
        [-2.8400, -0.7849, -1.4096]], <-- 5번 단어의 벡터
        requires_grad=True)
```
* **행(Row)의 개수**: 총 6줄입니다. 사전의 크기(`vocab_size=6`)와 똑같습니다. 사전에 있는 모든 단어가 각자의 줄(좌표)을 하나씩 배정받은 것입니다.
* **열(Column)의 개수**: 총 3칸입니다. 우리가 설정한 차원 수(`output_dim=3`)와 동일합니다.
* **랜덤 값**: 안에 들어있는 숫자들은 0.3374처럼 아무 의미 없는 무작위(Random) 숫자들입니다. 아직 모델이 태어난 직후라서 단어의 뜻을 전혀 모르기 때문입니다.
* **`requires_grad=True`의 의미**: 이 랜덤 숫자들은 고정된 것이 아니라, 5장에서 본격적인 훈련을 거치며 정답을 맞히기 위해 스스로 최적의 숫자로 변해간다는(미분되어 업데이트된다는) 뜻입니다.

## 3. 🖼️ 임베딩은 사실 거대한 '룩업 테이블(Lookup Table)'이다 (Figure 2.16)

<img width="646" height="390" alt="Figure 2 16" src="https://github.com/user-attachments/assets/2451021c-1497-4466-a8e4-1462ac626045" />


이제 만들어진 임베딩 계층에 실제 단어 ID를 집어넣어 보겠습니다.

### 📝 코드 2: 단어 1개 집어넣기
```python
print(embedding_layer(torch.tensor([3])))
```
* **출력**: `tensor([[-0.4015,  0.9666, -1.1481]])`
* 출력된 값을 위에 있는 가중치 행렬과 비교해 보세요. 정확히 **4번째 줄(인덱스 번호 3)**에 있는 숫자 3개가 그대로 튀어나왔습니다.

### 📝 코드 3: 여러 개의 단어(문장) 집어넣기
```python
print(embedding_layer(input_ids)) # input_ids는 [2, 3, 5, 1] 입니다.
```
* **출력**:
  ```text
  tensor([[ 1.2753, -0.2010, -0.1606],  <-- 2번 인덱스의 행
          [-0.4015,  0.9666, -1.1481],  <-- 3번 인덱스의 행
          [-2.8400, -0.7849, -1.4096],  <-- 5번 인덱스의 행
          [ 0.9178,  1.5810,  1.3010]]) <-- 1번 인덱스의 행
  ```
* **결론**: 그림 2.16이 말하고자 하는 핵심이 바로 이것입니다. 파이토치의 `Embedding` 계층은 복잡한 연산을 하는 것이 아니라, 그저 입력된 숫자표(ID)를 보고 거대한 서랍장(가중치 행렬)에서 **해당 층의 서랍을 쏙 빼서 가져오는 단순한 찾기 작업(Lookup Operation)**을 수행할 뿐입니다.

### 💡 [심화 지식] 원핫 인코딩(One-hot Encoding)과의 관계
전통적인 기계학습에서는 3번 단어를 `[0, 0, 0, 1, 0, 0]` 이라는 원핫 벡터로 만든 뒤, 방금 본 6x3 가중치 행렬과 곱하기(Matrix Multiplication)를 수행했습니다. 수학적으로 이 행렬 곱의 결과는 정확히 4번째 줄을 빼오는 것과 100% 동일합니다.
하지만 행렬 곱은 매우 무거운 연산입니다. 파이토치는 굳이 0을 곱하며 연산력을 낭비하지 않고, 메모리 주소에서 값만 바로 복사해오는 최적화된 룩업(Lookup) 방식을 사용하여 딥러닝 속도를 수백 배 이상 끌어올렸습니다.

## 4. 🌐 [2027년 최신 실무 트렌드] 기하급수적으로 팽창하는 임베딩 차원(Dimension)

GPT-3 시절만 해도 단어 하나를 표현하는 공간의 차원(`output_dim`)은 12,288차원이었습니다. 하지만 2027년 현재 실무에 배포되는 GPT-5 급, Llama 4 급의 거대 멀티모달 모델들은 임베딩 차원을 **16,384 차원에서 32,768 차원**까지 엄청나게 확장하여 훈련시킵니다.

* **차원이 커진다는 것의 실무적 의미**:
  단순히 숫자 배열이 길어지는 것이 아닙니다. 차원이 많을수록 단어가 짊어질 수 있는 '미묘한 뉘앙스(Contextual Nuance)'의 정보량이 폭발적으로 증가합니다.
  예를 들어 "사과"라는 단어가 있을 때, 차원이 작으면 단순히 [과일, 빨간색] 정도의 정보만 담지만, 차원이 수만 개로 늘어나면 [애플 회사, 스티브 잡스, 아이폰, 백설 공주, 뉴욕(Big Apple)] 등 문맥에 따라 변하는 상상 초월의 미세한 지식 뉘앙스를 모두 벡터 숫자들 안에 욱여넣을 수 있게 됩니다.
* 현대 AI의 경이로운 추론 능력은 바로 이 초기 랜덤 값들이 역전파를 통해 정교한 3만 차원의 별자리(벡터 공간)로 재배치되는 과정에서 탄생하는 것입니다.

### 💡 모범 답안 (정확한 이해를 위한 체크)

**답안 1**
정수형 토큰 ID는 단순히 딕셔너리 내의 순서표일 뿐, 단어와 단어 사이의 '의미적 거리'나 관계를 전혀 내포하고 있지 않습니다. (사과가 1, 강아지가 10이라고 해서 10배 차이 나는 것이 아님). 또한 신경망은 오차를 미분(역전파)하여 가중치를 미세하게 수정하는 연속적인 공간(Continuous space)에서의 학습이 필요한데, 정수는 불연속적이라 미분이 불가능합니다. 이를 실수형 다차원 벡터로 바꾸어야만 공간상에서 단어 간의 거리를 미세하게 좁히거나 넓히면서(학습하면서) 풍부한 의미를 담아낼 수 있습니다.

**답안 2**
임베딩 계층은 `vocab_size` 개의 행과 `output_dim` 개의 열을 가진 거대한 가중치 행렬(서랍장)을 메모리에 들고 있습니다. 토큰 ID `[3]`이 입력되면, 이 계층은 복잡한 수학적 연산을 수행하는 것이 아니라, 가중치 행렬의 0번 인덱스부터 세어 **정확히 4번째 행(Row)**에 저장된 실수 벡터값을 통째로 쏙 빼서(Lookup Operation) 반환합니다. 즉, ID표 번호에 맞는 서랍을 열어 내용물을 꺼내오는 단순 검색 작업입니다.

**답안 3**
수학적으로 임베딩 룩업은 단어를 `[0,0,0,1,0,0]`과 같은 **원핫 인코딩(One-hot Encoding)** 벡터로 만든 뒤 전체 가중치 행렬과 곱셈(Matrix Multiplication)을 하는 것과 결괏값이 100% 동일합니다. 하지만 사전의 크기가 5만 개(GPT-2)만 되어도 5만 칸 중 49,999칸이 0인 거대한 벡터를 만들고, 모든 0에 대해 무의미한 곱셈 연산을 수행해야 하므로 컴퓨팅 자원(GPU)과 메모리가 극도로 낭비됩니다. 반면 룩업 방식은 곱셈을 몽땅 생략하고 필요한 행의 주소로 찾아가 값만 복사해오기 때문에 딥러닝 속도가 기하급수적으로 빠릅니다.

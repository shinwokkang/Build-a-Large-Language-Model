## 1. 📂 텍스트 데이터 준비하기 (다운로드 및 읽기)

LLM을 학습시키려면 당연히 텍스트 파일이 필요합니다. 책에서는 에디스 워튼(Edith Wharton)의 단편 소설 "The Verdict"를 샘플 데이터로 사용합니다.

### 📝 코드 1: 인터넷에서 파일 다운로드하기
```python
import urllib.request

# 깃허브에 올라와 있는 원본 텍스트 파일의 주소
url = ("https://raw.githubusercontent.com/rasbt/"
       "LLMs-from-scratch/main/ch02/01_main-chapter-code/"
       "the-verdict.txt")
file_path = "the-verdict.txt"

# urllib 모듈을 이용해 해당 URL의 파일을 내 컴퓨터(file_path)로 다운로드
urllib.request.urlretrieve(url, file_path)
```
* **코드 해설**: 파이썬에 내장된 `urllib` 라이브러리를 사용하면 브라우저를 켜지 않고도 파이썬 스크립트 상에서 곧바로 인터넷의 파일을 다운로드할 수 있습니다.

### 📝 코드 2: 다운로드한 파일 읽어오기
```python
# 'r' (읽기 모드), 'utf-8' (글자가 깨지지 않도록 인코딩 지정)으로 파일 열기
with open("the-verdict.txt", "r", encoding="utf-8") as f:
    raw_text = f.read()

print("Total number of character:", len(raw_text))
print(raw_text[:99]) # 처음부터 99번째 글자까지만 출력
```
* **출력 결과**: 약 20,479개의 글자(character)로 이루어져 있음을 확인했습니다. 이제 이 거대한 하나의 문자열(`raw_text`)을 잘게 쪼갤 차례입니다.

## 2. ✂️ 토큰화의 진화 과정 (정규 표현식 `re` 모듈 활용)

가장 초보적인 방법은 띄어쓰기(공백)를 기준으로 문장을 자르는 것입니다. 하지만 쉼표나 마침표 처리가 문제입니다. 파이썬의 `re` (정규 표현식) 라이브러리를 사용하여 코드가 어떻게 진화하는지 보겠습니다.

### 📝 코드 3: 1단계 - 공백으로만 자르기
```python
import re
text = "Hello, world. This, is a test."
result = re.split(r'(\s)', text)
print(result)
```
* **결과**: `['Hello,', ' ', 'world.', ' ', 'This,', ' ', 'is', ' ', 'a', ' ', 'test.']`
* **문제점**: `Hello,` 가 통째로 하나의 단어로 잡혔습니다. 이렇게 되면 쉼표가 안 붙은 `Hello`와 별개의 단어로 취급되므로, 모델의 단어 사전이 꼬이게 됩니다. 구두점을 떼어내야 합니다.

### 📝 코드 4: 2단계 - 쉼표와 마침표 떼어내기
```python
# 정규식 '([,.]|\s)' : 쉼표나 마침표, 또는(|) 공백(\s)을 기준으로 자르라는 뜻
result = re.split(r'([,.]|\s)', text)
print(result)
```
* **결과**: `['Hello', ',', '', ' ', 'world', '.', '', ' ', 'This', ',', '', ' ', 'is', ' ', 'a', ' ', 'test', '.', '']`
* **해결 및 새로운 문제**: 구두점 분리에는 성공했지만, 리스트 중간중간에 `''` (빈 문자열)이나 `' '` (공백) 같은 쓸데없는 쓰레기 데이터가 껴있습니다. 이를 청소해야 합니다.

## 3. 🧹 대소문자 유지와 공백 제거 로직 완벽 분석

### 📝 코드 5: 3단계 - 불필요한 공백 제거하기 (List Comprehension)
```python
# 리스트 컴프리헨션 문법 적용
result = [item for item in result if item.strip()]
print(result)
```
* **결과**: `['Hello', ',', 'world', '.', 'This', ',', 'is', 'a', 'test', '.']`
* **코드 완벽 해설**:
  - `item.strip()` 함수는 문자열 양끝의 공백을 지워줍니다. 
  - 만약 `item`이 원래 빈 문자열(`''`)이거나 공백(`' '`)이었다면, `strip()`을 거치면 완전히 비어버리게 되어 `False`로 판정됩니다.
  - 즉, `if item.strip()` 조건문은 **"공백이나 빈 문자열이 아닌, 진짜 내용물이 있는 글자만 통과시켜라"**라는 뜻의 파이썬의 우아한 필터링 문법(리스트 컴프리헨션)입니다.

### 💡 실무적 핵심 포인트: 왜 `.lower()`로 소문자 통일을 안 했을까?
코드를 자세히 보면 텍스트를 모두 소문자로 바꾸는 `text.lower()`를 쓰지 않고 **대문자(Hello, This)를 그대로 살려두었습니다.**
1. **대소문자 유지 이유**: 대문자는 문장의 시작을 알리는 중요한 문법적 단서이자, 일반 명사(apple)와 고유 명사(Apple 회사)를 구분하게 해주는 핵심 정보입니다. LLM이 올바른 문장력을 배우려면 원본 그대로의 대소문자를 유지해야 합니다.
2. **공백 제거 이유**: 공백을 토큰으로 두면 메모리와 연산량이 낭비되므로 지웠습니다. **단, 파이썬(Python) 소스 코드를 학습시킬 때는 들여쓰기(공백) 자체가 문법이므로 절대로 지우면 안 됩니다.**

## 4. 🚀 최종 완성본: 모든 특수문자 처리와 실전 적용

### 📝 코드 6: 최종 토큰화 코드 (단편 소설에 적용)
```python
# 여러 종류의 구두점과 이중 대시(--)까지 모두 커버하는 궁극의 정규식
preprocessed = re.split(r'([,.:;?_!"()\']|--|\s)', raw_text)

# 불필요한 공백들 제거
preprocessed = [item.strip() for item in preprocessed if item.strip()]

print(len(preprocessed))
print(preprocessed[:30]) # 첫 30개 토큰 확인
```
* **코드 해설**: 정규식 `([,.:;?_!"()\']|--|\s)` 은 쉼표, 마침표, 콜론, 따옴표, 괄호 등 책에서 나올 수 있는 모든 기호, 그리고 영문 소설에서 자주 쓰이는 이중 대시(`--`), 그리고 공백(`\s`)을 모두 독립적인 분리 기준으로 삼겠다는 뜻입니다.
* **최종 결과**: 2만 자의 텍스트가 **4,690개의 깔끔한 토큰 리스트**로 완벽하게 변환되었습니다. 
* `['I', 'HAD', 'always', 'thought', 'Jack', 'Gisburn', 'rather', 'a', 'cheap', 'genius', '--', 'though', 'a', 'good', 'fellow', ...]` 
* 이제 이 쪼개진 글자 조각들에 '숫자 번호(Token ID)'표를 붙여줄 준비가 끝났습니다.
